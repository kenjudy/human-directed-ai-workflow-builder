# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Repo Is

The **pdca-scaffold** skill: a generalized PDCA skill generator for any complex repeatable human task that benefits from agentic Human-in-the-Loop (HITL) workflows. It uses 5-layer Socratic discovery to produce a valid, installable Claude skill for any domain.

- Master prompt files in `5. Scaffold/`
- Built into `claude-skill/pdca-scaffold.skill`
- Example generated skills in `scaffolded-skills/`

## Commands

**Build the skill:**
```bash
bash claude-skill/build-scaffold.sh
```
Reads `5. Scaffold/*.md`, generates `claude-skill/pdca-scaffold/references/`, zips into `claude-skill/pdca-scaffold.skill`.

**Install locally after build:**
```bash
unzip -o claude-skill/pdca-scaffold.skill -d ~/.claude/skills/
```

**Validate after build with skill-creator:**
```
/skill-creator
```
Verifies SKILL.md format, description triggering, and behavioral effectiveness before production use.

## Architecture

```
Master Prompts (Obsidian .md)  →  build-scaffold.sh  →  pdca-scaffold.skill
5. Scaffold/5a...md                                          ↓
5. Scaffold/5b...md          →  pdca-scaffold/references/  →  .skill zip
5. Scaffold/5c...md                                          ↓
                                                      Install to ~/.claude/skills/
                                                      or upload to claude.ai
```

**Never edit `claude-skill/pdca-scaffold/references/` directly** — they are build artifacts regenerated by `build-scaffold.sh`.

## Key Files

| File | Role |
|---|---|
| `5. Scaffold/5a. Socratic Discovery.md` | 5-layer Socratic question framework (source of truth) |
| `5. Scaffold/5b. Generation Templates.md` | Templates for generated skill files (source of truth) |
| `5. Scaffold/5c. Refinement Protocol.md` | Active learning loop for generated skills (source of truth) |

| `claude-skill/build-scaffold.sh` | Assembles pdca-scaffold.skill from masters |
| `scaffolded-skills/` | Example skills generated by pdca-scaffold |
| `.github/workflows/release.yml` | Triggered by `v*.*.*` tag push; publishes GitHub Release |

## Releasing

1. Note change in `CHANGELOG.md` (repo root)
2. Commit and push to main
3. `git tag v1.x.x && git push --tags`

The release workflow runs automatically and attaches `pdca-scaffold.skill` to the GitHub Release.

## Issue Tracking (Beads)

This project uses **beads (`bd`)** for issue tracking. Run `bd onboard` to get started.

```bash
bd ready                              # Find available work
bd show <id>                          # View issue details
bd update <id> --status in_progress   # Claim work
bd close <id>                         # Complete work (after human sign-off)
bd sync                               # Sync with git
```

## Session Completion (Mandatory)

Work is **not complete** until `git push` succeeds and the human has signed off on the Check and Act phases.

**Required sequence:**
1. File beads issues for any remaining work
2. Rebuild if prompt content changed: `bash claude-skill/build-scaffold.sh`
3. Close finished issues, update in-progress ones
4. Push:
   ```bash
   git pull --rebase
   bd sync
   git push
   git status  # must show "up to date with origin"
   ```
5. Hand off — provide context for next session

**NEVER** say "ready to push when you are" — push yourself. If push fails, resolve and retry.

## Human Supervision (Non-Negotiable)

- **STOP and present to the human** before committing, pushing, or closing issues
- Do not batch multiple steps without human confirmation between them

## Validating Prompt Changes

Any edit to master prompt files (`5. Scaffold/*.md`) requires:

1. Rebuild: `bash claude-skill/build-scaffold.sh`
2. Inspect the generated `claude-skill/pdca-scaffold/references/` files
3. Validate with `/skill-creator` if behavioral changes are significant
4. Run through at least one discovery session to confirm the flow is intact
