# ai-workflow-builder

A Claude skill that generates customized, installable AI workflow skills for any complex repeatable human task that benefits from agentic Human-in-the-Loop (HITL) workflows.

You describe your task. It uses 5 layers of Socratic questioning to understand it deeply, then generates a complete skill: SKILL.md with domain-specific triggering, working agreements with explicit human/AI ownership, PLAN/DO/CHECK/ACT phase prompts, and quality gates. A built-in refinement protocol sharpens the workflow after each cycle without letting it grow longer.

---

## Install

Download `ai-workflow-builder.skill` from the [latest GitHub Release](https://github.com/kenjudy/ai-workflow-builder/releases/latest).

**Claude.ai:**
Go to **Customize** → **Skills** → **Add Skill** → upload the file.

**Claude Code (local install):**
```bash
unzip -o ai-workflow-builder.skill -d ~/.claude/skills/
```

---

## How It Works

**Phase 1: Discovery** — 5 layers of structured questions uncover your task, how human and AI responsibilities divide, what quality looks like, where intervention is needed, and what you want to learn each cycle. A mandatory confirmation gate after Layer 5 is the primary HITL checkpoint.

**Phase 2: Generation** — A complete skill is generated in your project directory, version-controlled alongside the work it governs.

**Phase 3: Refinement (per cycle)** — After each ACT retrospective in the generated skill, the refinement protocol proposes specific diffs back to the skill's own reference files. You approve each change. Anti-drift rule: no more than +10 net lines per refinement cycle.

---

## After Generation

Before using a generated skill in production, validate it with `/skill-creator` in Claude Code:

- Verifies SKILL.md format and that the description triggers correctly
- Runs with-skill vs. without-skill eval comparisons to confirm behavioral effectiveness
- Surfaces improvements before the first real cycle

A generated skill that hasn't been through eval is an untested assumption.
